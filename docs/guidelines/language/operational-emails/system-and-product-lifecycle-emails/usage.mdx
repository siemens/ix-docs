## Subject lines

Include system or product lifecycle events, such as upcoming updates, directly in the subject line.

<div className="dos-and-donts">
<div className="dos">
<ul aria-label="Recommended practices">
    <li>[App name] | Version 2.0 available</li>
    <li>[App name] | End-of-life announcement for "legacy control panel model X-200"</li>
    <li>[App name] | Test Version 2.1 available for staging environment</li>
    <li>[App name] | Scheduled maintenance September 24, 2026, 01:00–13:00</li>
  </ul>
</div>
</div>

## Body text

### Version and release announcements

Include the new version number and release date.

<div className="dos-and-donts">
<div className="dos">
<ul aria-label="Recommended practices">
    <li>Version 2.1.0 will be available November 10.</li>
  </ul>
</div>
</div>

Add a summary of key changes or improvements.

<div className="dos-and-donts">
<div className="dos">
<ul aria-label="Recommended practices">
    <li>This release includes enhanced firewall rules and improved logging.</li>
  </ul>
</div>
</div>

Include installation or update instructions with clear links, buttons or next steps.

<div className="dos-and-donts">
<div className="dos">
<ul aria-label="Recommended practices">
    <li>Download the update via the [App name] portal and restart your system.<br/>
  Button: Open</li>
  </ul>
</div>
</div>

### End-of-life announcements

Include upgrade paths, migration guides and support options for product transitions.

<div className="dos-and-donts">
<div className="dos">
<ul aria-label="Recommended practices">
    <li>Our dedicated support team is available to assist you with your migration to the "Smart factory OS". Contact us at [email address] or visit our [Migration portal].</li>
  </ul>
</div>
</div>

Use absolute terms when describing the end of a product status or availability.

<div className="dos-and-donts">
<div className="dos">
<ul aria-label="Recommended practices">
    <li>Industrial Gateway V7.2 will no longer receive security updates after June 30, 2026.</li>
    <li>Be advised that support for "control panel X-200" will officially end on December 31, 2026. Migrate to "control panel Z-200" for continued support and enhanced features.</li>
  </ul>
</div>
</div>

### Release approvals

Include a clear request for approval.

<div className="dos-and-donts">
<div className="dos">
<ul aria-label="Recommended practices">
    <li>Review and approve release V2.1.0 by November 10, 2026.</li>
  </ul>
</div>
</div>

Use links or attachments for release notes.

<div className="dos-and-donts">
<div className="dos">
<ul aria-label="Recommended practices">
    <li>See attached release documentation for full details.</li>
  </ul>
</div>
</div>

### Staging and test releases

Include the test environment details.

<div className="dos-and-donts">
<div className="dos">
<ul aria-label="Recommended practices">
    <li>The staging release is now available at [web address].</li>
  </ul>
</div>
</div>

Use instructions for feedback or reporting issues.

<div className="dos-and-donts">
<div className="dos">
<ul aria-label="Recommended practices">
    <li>Report any anomalies via the quality portal.</li>
    <li>Send your feedback to [email address] by September 22, 2026.</li>
  </ul>
</div>
</div>

### Maintenance announcements

Include the maintenance window and impact.

<div className="dos-and-donts">
<div className="dos">
<ul aria-label="Recommended practices">
    <li>Scheduled maintenance will occur on Nov 15, 02:00–04:00. Temporary downtime expected.</li>
  </ul>
</div>
</div>

Use instructions for preparation or follow-up.

<div className="dos-and-donts">
<div className="dos">
<ul aria-label="Recommended practices">
    <li>Ensure backups are completed before the maintenance window.<br/>
  Button: Open Drive manager</li>
  </ul>
</div>
</div>
